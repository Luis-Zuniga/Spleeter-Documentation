Spleeter Dependencies:

  • absl-py:
    Tensorflow will not correctly import without it. It is a collection of libraries for building python applications. Seems to be derived from Abseil, which was a C++ library that adds functionality beyond the standard library. Unsure what it does specifically in Spleeter.
  • astor:
    A library used to allow easy manipulation of Python source via the AST (Abstract Syntax Tree).
    https://pypi.org/project/astor/
  • audioread:
    Decodes audio files using ffmpeg or whatever backend is available (alternatives include Gstreamer, Core Audio, MAD, etc.).
    https://pypi.org/project/audioread/
  • cffi:
    Stands for "C Foreign Function Interface." Used to interact with C code.
    https://cffi.readthedocs.io/en/latest/
  • decorator:
    Used for establishing decorators in python. Decorators take in a function, it adds functionality (i.e. more code gets executed related to the input function), and then returns it.
    https://pypi.org/project/decorator/
  • ffmpeg-python:
    Wrapper/Bindings for interacting with ffmpeg, a multimedia framework.
  • future:
    Bridges compatability between Python 2 and Python 3 when using Python 3 codebase.
    https://pypi.org/project/future/
  • gast
    Bridges compatability between the AST (Abstract Syntax Tree) of various python versions.
    https://pypi.org/project/gast/
  • google-pasta
    Enables the refactoring of python source code through AST (Abstract Syntax Tree) modifications.
    https://github.com/google/pasta
  • grpcio
    Unable to find description at the moment.
    https://github.com/grpc/grpc/tree/master/src/python/grpcio
  • h5py
    Pythonic interface for the HDF5 binary data format. Useful for storing large amounts of numberical data and manipulating the data from NumPy.
    https://www.h5py.org/
  • importlib-metadata
    Provides access to installed package metadata, usually for third-party packages installed via Pip and other means.
    https://docs.python.org/3/library/importlib.metadata.html
  • importlib-resources
    This is a backport of the importlib.resources module for Python 2.7 & 3.4 to 3.8. It is useful for people running Python 3.9.
    https://pypi.org/project/importlib-resources/
  • joblib
    Set of Tools to provide lightweight pipelining in Python, like simple parallel computing and transparent disk-caching of function and lazy re-evaluation. Meant to be fast.
    https://joblib.readthedocs.io/en/latest/
  • keras-applications
    Module of the Keras deep learning library that provides model definitions and pre-trained weights for many popular architectures, such as VGG16, ResNet50, Xception, MobileNet and others. Keras is built upon the Tensorflow library.
    https://pypi.org/project/Keras-Applications/
    https://keras.io/
  • keras-preprocessing
    Data preprocessing and data augmentation module of the Keras deep learning library. Utilities are provided for working with image data, text data and sequence data. Keras is built on the Tensorflow library.
    https://pypi.org/project/Keras-Preprocessing/
    https://keras.io/
  • librosa
    Library for music and audio analysis. It's used to create music information retrieval systems.
    https://pypi.org/project/librosa/ http://librosa.github.io/librosa/
  • llvmlite
    Lightweight LLVM (Low Level Virtual Machine) python binding for writing JIT (Just-In-Time) compilers.
    https://pypi.org/project/llvmlite/
